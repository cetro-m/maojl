## 概览

- 搭建了含 12 个 Collection Type 的 Strapi v5 CMS，覆盖产品、新闻文章、可下载资源、首页、关于页和联系表单。
- 构建了 Nuxt 4 双语前端（英文/中文），支持基于域名切换语言、Strapi 内容代理和服务端认证。
- 交付了完全品牌化的 Admin 后台，附带自研分类树筛选和拖拽排序插件。
- 集成了 CKEditor 5 富文本编辑器、上传自动转 WebP 响应式图片、软删除与可配置定时清理。
- 实现了 Nitro 服务层，负责认证会话、文件下载与预览权限控制、联系表单 SMTP 通知、速率限制和安全响应头。

## CMS — Strapi 后端

**内容架构**

十二个 Collection Type 构成内容骨架：产品带多级分类树，新闻带分类和 Banner，资源带分类文件库，首页和关于页 Banner，联系页面，以及通过生命周期钩子发送邮件通知的联系表单收件箱。

两个自研插件扩展了 Strapi Content Manager：

- 树状分类筛选器：编辑者无需离开列表页，即可按分层分类树筛选产品、文章和资源列表。
- 拖拽分类排序器：在分类树中持久化自定义排序，排序结果直接用于前端侧边栏展示。

第三方插件提供了上传时自动生成 WebP 响应式图片（`strapi-plugin-img-convert`）、CKEditor 5 富文本编辑器、以及带生命周期钩子和可选定时清理的软删除。

**Admin 后台定制**

后台管理界面完成了全套品牌化改造：自定义公司 Logo、Favicon，以及贯穿按钮、链接、边框和底色的一套蓝色主色调系统。

完整的国际化覆盖英文和简体中文：

- 所有 Admin 界面文本通过自定义 locale 文件翻译。
- Content Manager 列表页和编辑页中的硬编码英文字符串在运行时替换为中文。
- 内容模型的字段标签根据所选 Admin 语言显示。
- 全局搜索框的占位符跟随当前语言。
- 翻译缺失收集器可在调试模式下运行，帮助开发阶段发现未翻译的 key。

Content Manager 导航被重新分组：内容类型归入带标签的分区（产品、新闻、页面、系统），每个分区内的 Collection Type 顺序按编辑工作流排列。

文档标题根据当前路由动态更新。一个 DOM 观察器确保 Vite/HMR 热更新后，标题替换、按钮标签、语言切换标记和导航分组标题依然正确。

产品与文章的编辑页面增加了"前端查看"按钮。编辑者一键即可在新标签页中打开对应的线上页面，路径基于该内容的分类 slug、文档 ID 和语言生成。

## Web — Nuxt 前端

**技术栈**

基于 Nuxt 4、Nuxt UI 4、Tailwind CSS 4、daisyUI 5 和 `@nuxtjs/i18n` 构建。自托管 Inter 字体，WOFF2 格式覆盖 400–900 字重。

**公共页面**

- **首页** — 公司着陆页，包含 Banner、推荐产品和新闻摘要。
- **产品** — 分类侧边栏导航、产品卡片列表、详情页（概览/规格/图库/下载 Tab）、响应式图片画廊。
- **新闻** — 按分类筛选的文章列表、CKEditor 富文本详情页。
- **资源** — 可筛选的资源库，含登录门控的文件预览与下载。
- **关于** — 公司介绍页，含荣誉墙卡片、触摸滚动和键盘导航支持。
- **联系** — 联系表单，含客户端验证和服务端提交。
- **搜索** — 客户端全文搜索，覆盖产品、新闻和资源，支持多词匹配、加权字段评分、类型/分类筛选、分页和 URL 查询状态持久化。
- **认证** — 登录、注册、隐私政策和服务条款页面。

**服务端层（Nitro）**

自定义 Strapi 公共 API 代理（`/api/strapi/**`）限制访问已知的公共 Collection 根路径，对查询参数做深度、嵌套和大小消毒，实施每 IP 速率限制，并设置可配置 `s-maxage` 的缓存控制头。

认证使用 httpOnly JWT Cookie。会话状态通过服务端端点刷新；未登录用户返回 `{ user: null }`，不会暴露 401。登录、注册、登出和会话端点遵循同一套基于 Cookie 的流程。

资源和产品的文件下载与预览通过 Nitro 路由流式传输，浏览器不会接触到原始 Strapi 媒体 URL。预览仅限浏览器安全类型（PDF, PNG, JPG, JPEG, WebP）。访问门控检查用户认证状态、资源可见性以及可选的邮箱白名单。

联系表单提交在服务端做校验，经过内存速率限制器检查后转发给 Strapi 创建新的 `contact-message` 条目，并通过生命周期钩子向配置的收件人发送邮件通知。

安全响应头添加了点击劫持保护、MIME 嗅探防护、Referrer 策略、权限策略和基线 CSP。接受数据的端点限制请求体大小、通过 Origin 和 Fetch Metadata 检查拒绝跨站请求、且只接受 JSON。

产品与新闻内容中的 CKEditor HTML 在渲染前经过严格属性白名单消毒。脚本、事件处理器、不安全 URL、任意 CSS 函数、嵌入内容和表单控件被移除，同时保留常见的编辑样式（对齐、安全字号/字体、颜色、响应式图片宽度、表格尺寸调整）。

## 改进

- Strapi 代理的查询参数校验仅允许 Strapi API 文档定义的顶层 key（`fields`, `filters`, `locale`, `pagination`, `populate`, `publicationState`, `sort`），防止注入任意查询参数。
- 文件下载路由使用流式传输而非将整个文件读入内存再发送。
- 搜索在客户端本地索引产品、新闻和资源，无需依赖 Meilisearch 等外部搜索服务。
- Sitemap 路由从 Strapi 获取动态产品与新闻详情 URL，后端不可达时自动回落为静态条目。
- 服务启动前执行环境变量校验，提前捕获缺失的密钥（`APP_KEYS`, `API_TOKEN_SALT`, `ADMIN_JWT_SECRET`, `TRANSFER_TOKEN_SALT`, `JWT_SECRET`, `ENCRYPTION_KEY`, `PUBLIC_URL`, `DATABASE_CLIENT`）。

## 修复

- Strapi Admin Vite 开发服务器的 host 已显式列入白名单，防止使用隧道服务时出现 EPERM 和 host-check 错误。
- Strapi 本地化报告改为按需生成，不再以仓库快照形式提交。
- 未到发布日期的草稿条目不再出现在公共 API 响应中——公共代理请求将 `publicationState` 显式设为 `live`。
- CKEditor 渲染的过宽表格已做换行处理，防止移动端横向溢出。
- 认证会话端点对游客不报错——返回 `{ user: null }` 而非浏览器可见的 401。
- Strapi 代理请求的分页参数上限为每页 100 条，防止意外的大载荷。
- 通过软删除插件配置，已软删除的条目默认不出现在公共 API 响应中。
