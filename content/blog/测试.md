## 为什么先做内容模型

个人博客很容易在一开始就陷入视觉风格的选择。

一个有记忆点的界面当然重要，但更重要的是：这个网站能不能长期支撑日常写作、项目记录和知识整理。

这一版博客先把长文章、短笔记和结构化数据分开处理。长文章用来沉淀相对完整的观点，短笔记用来记录现场观察，结构化数据则可以在后续继续服务首页、归档页、人物页、项目页或简历页。

::callout{type="terminal" title="先稳定，再变复杂"}
第一个里程碑不是动画，也不是复杂交互，而是从 Markdown 到可查询、可校验、可渲染页面的稳定路径。
::

## Nuxt Content 在这里做什么

在这个项目里，Nuxt Content 主要负责三件事：

- 把 Markdown 内容变成可以查询的数据。
- 用 schema 校验每篇文章的字段格式。
- 在普通 Markdown 不够用时，允许插入 MDC 组件。

```ts [content.config.ts]
blog: defineCollection({
  type: 'page',
  source: 'blog/**/*.md',
  schema: articleSchema,
})
